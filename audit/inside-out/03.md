## Patterns
- shadcn/ui convention: primitives in `ui/`, customized in `components/` subfolders
- Feature components are co-located with their hooks and services
- Page components are thin wrappers that compose feature and shared components
- Modal naming convention: `*Modal.tsx` for dialogs, `*Sheet.tsx` for side sheets, `*Drawer*.tsx` for drawers
- Cell renderer pattern: `*CellRenderer.tsx` components for AG Grid custom cell rendering
- Skeleton pattern: `*Skeleton.tsx` components for loading states (IntegrationCardSkeleton, TemplateCardSkeleton)

## Undocumented
- `multi-select.tsx` in shadcn/ui is a custom addition (not part of standard shadcn)
- `sidebar.tsx` in shadcn/ui exists but the app uses a custom `AppSidebar.tsx` instead
- `InfinityLoop.tsx` and `InfinityLoader.tsx` are custom branded loading animations
- The grid feature has 30+ components, making it the most complex feature module
- 8 export flow components exist under `features/grid/components/actions/flows/` for different platform exports

## Concerns
- `src/pages/Index.tsx` exists but is never referenced in routing (dead file)
- The grid feature's 30+ components may benefit from further sub-modularization
- Settings page has 9 tab components; each is a separate file with no shared tab abstraction
- `ProtectedRoute.tsx` exists in `components/auth/` but the actual route protection is in `AppLayout.tsx` via `useAuth()` check, not via a ProtectedRoute wrapper component

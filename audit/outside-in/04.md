# 04 - Routing

## What Docs Say

### Route Configuration

All routes are defined in `src/App.tsx`. CLAUDE.md states there are **14 page routes**, all lazy-loaded via `React.lazy` + `<Suspense>`.

### Auth Guards

- `ProtectedRoute` component exists in `components/auth/` for auth gating
- No permission-gated routing documented (old app had `PrivateRoute` with `requiredPermission` prop)
- All authenticated routes accessible to all users regardless of workspace role

### Lazy Loading

All page components use `React.lazy` + `<Suspense>`. New pages must follow the same pattern. Synchronous imports for page components are prohibited.

### Navigation Patterns

- **Template activation**: navigates to `/datasets/:folderId/:listId/original?mappingId=xyz` with React Router state (`templateMapped`, `templateId`, `mappingId`)
- **Workspace switch**: navigates to `/` (home), forces full data reload
- **Template pending flow**: uses `sessionStorage.pendingTemplateId` to survive navigation to `/datasets?createnewlist=true` and back to `/templates`

### URL Parameters

| Route | Query Params | Purpose |
|-------|-------------|---------|
| `/datasets/:folderId/:listId/original` | `mappingId` | Template context for column logos |
| `/datasets` | `createnewlist=true` | Auto-open create list modal |
| `/auth` or `/login` | `partner=CODE` | Partner referral code |
| `/linkedin-analytics` | `audienceId`, `connectedAccountId` | Analytics context |

### Template Link Handler

A `TemplateLinkHandler` page exists at an undocumented route that handles `GET /welcome/initialize-template?email=` for welcome email template initialization.

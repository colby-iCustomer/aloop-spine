## Patterns
- HSL color system with CSS variables (shadcn standard)
- Blue-branded color scheme (213 hue family throughout)
- Card family system with consistent sizing variables
- PLG (Product-Led Growth) card naming convention
- Responsive grid via CSS Grid auto-fill with CSS variable min-widths
- Staggered entrance animations on grid children (nth-child delays)

## Undocumented
- The `.mosaic-grid` class suppresses all CSS animations on children with `!important` to prevent conflicts with Framer Motion
- AG Grid scrollbar CSS is split between `theme.ts` (Quartz params) and `index.css` (manual overrides for corners, buttons, pinned columns)
- The `--qa-gradient` variable creates a three-stop gradient for the Quick Actions panel
- `.plg-action-tilt` applies a subtle rotation only during auto-cycling, not on manual hover
- Body background is a fixed gradient, not a flat color

## Concerns
- The index.css file is 617 lines, mixing global resets, component classes, AG Grid overrides, and TipTap styles in a single file
- Two competing scrollbar strategies: global thin scrollbars (4px) vs AG Grid wider scrollbars (via theme params)
- AG Grid CSS uses many `!important` overrides which makes the cascade fragile
- The `next-themes` dependency is imported but dark mode toggle functionality is not visible in the UI code examined (no theme toggle button found in AppSidebar or TopBar)
- Some card classes (`.plg-card`, `.plg-card-lift`) use `@apply` which compiles away at build time but makes the CSS harder to trace at runtime

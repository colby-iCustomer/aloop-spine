# 00 - Product Context (Inside-Out Audit)

## What Code Shows

Audience Loop (ALoop) is a **B2B audience data management platform** built for marketers and sales teams. From the code, the product does the following:

**Core capability: List-based data management**
- Users organize data into **Folders** (called "projects" in the API) containing **Lists** (datasets)
- Lists are spreadsheet-like grids powered by AG Grid Enterprise with full cell editing, copy/paste, column management
- Data enters via CSV upload, Google Sheets import, HubSpot import, Salesforce import, Apollo/ICP prospecting, Klaviyo import, or webhooks

**Template system (major feature, actively developed)**
- 47 marketing templates spanning prospecting, enrichment, outreach, analytics, ABM, lookalikes, and more
- Templates define typed field schemas that map onto list columns
- Templates drive AI agent enrichment (each field can have an `agentPrompt` for auto-fill)
- Template categories: prospecting, enrichment, outreach, analytics, management, lookalike, events, community, newsletter, social, retargeting, content, conversion, product, abm, suppression, seed-audiences, segmentation

**AI enrichment engine**
- Per-row and batch enrichment via agent prompts
- Fill column formula capability
- Enrichment job tracking with socket-based real-time progress
- Credit-based billing for enrichment operations

**Data quality operations**
- Email validation (ZeroBounce integration implied by credits-status endpoint)
- Data normalization (contact and account entity types)
- Deduplication (find duplicates by criteria field, merge with field-level control)

**Multi-platform audience sync (Live Audiences)**
- Upload audiences to LinkedIn Ads and Meta Ads
- Sync audiences, attach to campaigns/ad sets
- SmartLead integration for outreach
- Google Ads, Reddit Ads, Mailshake, Klaviyo export flows

**Workspace and team management**
- Multi-workspace support with role-based access (owner, admin, editor, viewer)
- Team invitation system
- API key management per workspace
- Subscription/credits/billing via Stripe

**User problems it solves:**
1. Centralizing prospect and audience data from multiple sources
2. Enriching incomplete contact data via AI agents
3. Validating and normalizing contact information
4. Building and syncing audiences to ad platforms
5. Managing outreach sequences with personalized content
6. Deduplicating contact lists

## Patterns
- Product is clearly a rewrite/successor to an older frontend called "Precise" (multiple comments reference "old FE", "old app", "dual-FE support")
- The `/auth` redirect component explicitly maps old FE partner links to new FE paths
- Many service methods have comments like "Mirrors the old frontend" or "Ported from old repo"
- Template system is the primary differentiator; templates are the unit of product value

## Undocumented
- There is a dev-only `/dev/style-guide` route (conditionally rendered when `import.meta.env.DEV`)
- A `dev-bypass-token` mechanism exists in AuthContext that auto-authenticates with a mock user
- The product has an MCP (Model Context Protocol) auth flow with dedicated `/mcp-auth` and `/mcp-signup` pages
- Microsoft OAuth is supported alongside Google OAuth
- There is a "Pixels & Tags" feature suggesting website tracking/retargeting capability
- A `/welcome/template` route exists for handling template deep links (TemplateLinkHandler)

## Concerns
- The `audiencesService` in `src/api/services.ts` is fully deprecated (all methods throw errors) but still exported and has React Query hooks wrapping it
- `src/pages/Index.tsx` exists alongside the HomePage route at `/` but is not referenced in App.tsx routing
- The product has significant complexity for a frontend: 326 source files, 47 templates, 12 integration providers, 20+ socket events

### Route Guard Implementation
- **No ProtectedRoute wrapper** used despite `ProtectedRoute.tsx` existing
- Auth check lives in `AppLayout.tsx:13-17`: checks `isAuthenticated`, shows loader during `isLoading`, redirects to `/login` when unauthenticated
- `WorkspaceProvider` acts as a secondary gate: blocks rendering until workspace is resolved (shows loading/error UI)
- Protected routes are all children of the `<Route element={<NovuProviderWrapper><AppLayout /></NovuProviderWrapper>}>` layout route

### Redirects
1. `/auth` -> `/signup?partnercode=CODE` (if partnercode present) or `/login` (otherwise) - via `AuthRedirect` component
2. `/datasets/:projectId/:datasetId/dedupe` -> `/datasets/:projectId/:datasetId/original` - via `DedupeRedirect` component

### Lazy Loading
All 34 page components are lazy-loaded via `React.lazy()` with a single shared `Suspense` fallback (centered spinner).

## Patterns
- Single-level route structure (no nested layouts beyond AppLayout)
- All protected routes share the same layout (sidebar + topbar)
- Legacy route support via redirect components
- URL params convention: `:projectId` = folder ID, `:datasetId` = list ID, `:listId` = list ID (inconsistent naming)
- Analytics pages are separate routes per platform (LinkedIn, Meta, SmartLead) rather than parameterized

## Undocumented
- `/dev/style-guide` route is conditionally registered only when `import.meta.env.DEV` is true
- `/welcome/template` is a deep link handler (TemplateLinkHandler), likely used for marketing/onboarding links
- `/mcp-auth` and `/mcp-signup` routes exist for Model Context Protocol integration, a separate auth flow
- The NovuProviderWrapper wraps only the AppLayout route, meaning notification context is not available on public pages

## Concerns
- URL param naming inconsistency: `:projectId` and `:folderId` refer to the same concept (a folder/project), and `:datasetId` and `:listId` also refer to the same entity
- `ProtectedRoute.tsx` component exists in `src/components/auth/` but is NOT used anywhere in routing; auth protection is in AppLayout instead
- `src/pages/Index.tsx` exists but is never referenced in any route
- The dedupe redirect suggests the deduplication feature was once a standalone page but is now a side panel in ListViewPage
- Three separate analytics pages (LinkedIn, Meta, SmartLead) with no shared analytics route pattern

# 09 - Permissions & RBAC (Inside-Out Audit)

## What Code Shows

### Role System

**Defined roles** (`src/lib/permissions.ts:18-23`):
| Role | Label | Description |
|---|---|---|
| owner | Owner | Full control of workspace |
| admin | Admin | Full access to workspace settings and users |
| editor | Editor | Can view and edit lists |
| viewer | Viewer | Read-only access |

**Type definition** (`src/api/types/auth.ts:7`):
```typescript
export type UserRole = "owner" | "admin" | "editor" | "viewer";
```

**Legacy alias**: `member` is mapped to `editor` via `normalizeRole()` for backwards compatibility with older workspaces.

### Assignable Roles
Only three roles can be assigned to other users (`src/lib/permissions.ts:29-33`):
- admin, editor, viewer

The `owner` role is excluded from assignment (ownership is transferred, not assigned).

### Permission Check Hook

**useHasPermission** (`src/hooks/use-has-permission.ts`):
1. Gets current user from AuthContext
2. Gets current workspace from WorkspaceContext
3. If workspace has a `permissions` array (from API), checks if permission is included
4. Falls back to role-based lookup in `ROLE_PERMISSIONS` map if permissions array is empty

This means the backend can override the frontend's role-based defaults by returning a permissions array per workspace.

### Role Normalization

**normalizeRole()** (`src/lib/permissions.ts:92-97`):
- Maps `member` -> `editor`
- Maps unknown/null/undefined -> `editor`
- Valid roles pass through unchanged

### Where Permissions Are Checked

Observed permission checks in the codebase:
- **AppSidebar**: `useHasPermission("folder.view")`, `useHasPermission("folder.create")`, `useHasPermission("folder.edit")`, `useHasPermission("folder.delete")`
- **SettingsPage**: Filters tabs using `SETTINGS_TAB_ROLES` map against user role
- **Various components**: Import `useHasPermission` for conditional rendering

### Workspace Scoping

All data is workspace-scoped:
- `workspace-id` header injected on every API request via interceptor
- WorkspaceContext resolves active workspace before any data loads
- Switching workspace clears all React Query cache
- Workspace role comes from the workspace object, not the user object

### Auth Flow Protection

- **Route level**: `AppLayout.tsx` checks `isAuthenticated` and redirects to `/login`
- **Data level**: All API calls include JWT token; 401 triggers token refresh
- **Workspace level**: `WorkspaceProvider` blocks rendering until workspace resolves
- **No route-level role checks**: All role/permission checks happen at the component level

## Patterns
- Centralized permission definitions in a single file
- Hook-based permission checks (not HOCs or route guards)
- Backend can override frontend permissions via workspace.permissions array
- Role normalization handles legacy data gracefully
- Settings tabs use a whitelist approach (list allowed roles per tab)

## Undocumented
- The `member` role is a legacy alias that the backend may still return; it maps silently to `editor`
- `credits.allocate` is an owner-only permission not used visibly in the examined UI code
- `tutorial.manage` is only for owner and admin, suggesting content management capability
- The Workspace object carries both `role` and `permissions[]`; the hook prefers `permissions[]` when available
- `ProtectedRoute.tsx` exists but is not used; route protection is in AppLayout

## Concerns
- **No feature gating**: `useFeatureCheck` always returns true; no per-plan or per-workspace feature restrictions exist on the frontend
- **No route-level role guards**: A viewer can navigate to any protected route URL; restrictions are only at the component/button level
- **Viewer can see billing**: The settings tab visibility table shows viewers can access billing and usage tabs, which may expose sensitive plan/cost information
- **No audit trail**: Permission checks are purely UI-side with no logging; the backend is the actual enforcement layer
- **Role fallback to editor**: Unknown roles default to `editor` rather than the more restrictive `viewer`, which is a permissive default

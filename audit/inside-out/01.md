## Patterns
- shadcn/ui component system (Radix primitives + Tailwind + CVA)
- SWC used instead of Babel for React compilation (faster builds)
- Two toast systems in parallel: Radix-based toaster AND Sonner
- `next-themes` used for dark mode despite this being a Vite app (not Next.js)
- `lovable-tagger` in devDependencies indicates the project was initially scaffolded or developed using the Lovable platform

## Undocumented
- The `lovable-tagger` plugin is conditionally loaded only in development mode (`vite.config.ts:34`)
- AG Grid Enterprise requires a license key (not visible in code; presumably in env vars or configured server-side)
- TipTap rich text editor is used specifically for the column formula/prompt editor in the grid feature (`src/features/grid/components/TiptapColumnEditor.tsx`)

## Concerns
- `next-themes` dependency is unusual for a non-Next.js app; it works but is an unnecessary coupling to the Next.js ecosystem
- Two toast notification systems (Radix toast + Sonner) means developers must choose which to use; both are rendered in App.tsx
- `@typescript-eslint/no-unused-vars` is turned off, which means dead imports and variables won't be caught
- `noImplicitAny: false` and `strictNullChecks: false` in tsconfig weaken type safety significantly
- `noUnusedParameters: false` and `noUnusedLocals: false` further reduce TypeScript strictness

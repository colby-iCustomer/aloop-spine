# 08 - Design System

## What Docs Say

### Token System (`src/index.css`)

All design values live as CSS custom properties in `:root`. Never hardcode hex or HSL values in JSX or component files.

**Semantic Tokens (theme-aware, values change in `.dark`):**

| Token | Role |
|-------|------|
| `--background` | Page background |
| `--foreground` | Default text |
| `--card` / `--card-foreground` | Card surfaces (pure white) |
| `--primary` / `--primary-foreground` | Brand blue (`#2085fc`) |
| `--secondary` / `--secondary-foreground` | Light blue tint |
| `--muted` / `--muted-foreground` | Subdued surfaces and text |
| `--border` | Default border color |
| `--ring` | Focus ring color (lightened to reduce visual weight) |
| `--success` | Green for status/badges (dark green, not a wash) |
| `--warning` | Amber for pending/processing states |
| `--destructive` | Red for errors |
| `--radius` | Border radius base (`0.625rem`) |
| `--sidebar-*` | Sidebar-specific tokens |

**Palette Primitives (fixed brand colors):**

| Token | Value | Use |
|-------|-------|-----|
| `--accent-blue` | `213 64% 45%` (`#2966bc`) | Category/icon accents |
| `--accent-purple` | `271 91% 65%` (`#a855f7`) | Category/icon accents |
| `--accent-teal` | `172 76% 40%` (`#14b8a6`) | Category/icon accents |

**Panel-Specific Tokens:**

| Token | Value | Use |
|-------|-------|-----|
| `--gs-bg` | `138 76% 97%` | Getting Started panel background |
| `--gs-border` | `141 54% 84%` | Getting Started panel border |
| `--gs-item-border` | `141 54% 91%` | Getting Started step item borders |

**Naming Rule:** Semantic tokens by role (`--primary`, `--success`). Palette primitives by color (`--accent-blue`). Never name palette primitives by role.

### Background Gradient

```css
background: linear-gradient(145deg, hsl(213, 25%, 98.5%) 0%, hsl(220, 10%, 97.5%) 100%);
background-attachment: fixed;
```
Left side: barely-blue tint. Right side: near-neutral gray. Keep right endpoint low-saturation.

### Typography Hierarchy

| Level | Class | Use |
|-------|-------|-----|
| L1 - Page Title | `text-3xl font-bold text-foreground` | PageHeader only, one per page |
| L2 - Card/Section Title | `text-base font-semibold text-foreground` | Card headers, section titles |
| L3 - Item Title | `text-sm font-medium text-foreground` | Card item labels, list names |
| L4 - Description | `text-xs text-muted-foreground` | Help text, subtitles, metadata |

### Border Radius

Base is `--radius: 0.625rem`. shadcn components derive from this. Intentionally tighter than shadcn default. Do not increase.

### Focus Rings

`--tw-ring-offset-width: 0px` set globally. `--ring` lightened (`213 60% 72%`) to reduce visual weight. Do not revert.

### Quick Actions Gradient (HomePage)

`--qa-gradient`: `linear-gradient(135deg, hsl(213 78% 96%) 0%, hsl(228 68% 94%) 50%, hsl(245 60% 93%) 100%)` - saturated blue to indigo.

### Color Usage Rules

- Always use tokens. If color not in token system, add to `src/index.css` first.
- Tailwind: `bg-primary`, `text-success`, `border-border`
- CSS variables not in Tailwind: `bg-[hsl(var(--gs-bg))]` or inline `style`
- Per-element accent colors: pass CSS variable name as string, resolve at render time
- Never: `bg-[#2966bc]`, `style={{ color: '#f0fdf4' }}`, hardcoded HSL strings

### Known Violations (tracked in styling decisions)

**Fixed:**
- HomePage card titles: `text-slate-800` -> `text-foreground`
- HomePage card descriptions: `text-slate-600` -> `text-muted-foreground`
- HomePage arrow icon: `text-slate-400/600` -> `text-muted-foreground/text-foreground`

**Remaining:**
- TemplatesPage template button: `text-[13px]` should be `text-sm`
- FolderPage source pill: `text-[11px]` should be `text-xs`
- HomePage stat values: `text-blue-600`, `text-green-600` should use semantic tokens

### Dark Mode

Tokens support `.dark` class variant (documented in token system), but no explicit dark mode documentation. Dark mode values change via CSS custom properties.

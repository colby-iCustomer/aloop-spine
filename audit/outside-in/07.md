# 07 - Integrations

## What Docs Say

### Integration Categories

**OAuth-based Integrations (6 providers):**

| Provider | OAuth URL Pattern | Callback | Post-Connect Action |
|----------|------------------|----------|---------------------|
| LinkedIn Ads | `[GW]/api/integrations/linkedin-ads/auth?userId=&workspaceId=` | postMessage + polling | Airbyte setup |
| Meta Ads | `[GW]/api/integrations/meta-ads/auth?userId=&workspaceId=` | postMessage + polling | Airbyte setup |
| Reddit Ads | `[GW]/api/integrations/reddit-ads/auth?userId=&workspaceId=` | postMessage + polling | None |
| HubSpot | `[GW]/api/integrations/hubspot/auth?userId=&workspaceId=` | postMessage + polling | None |
| Google Sheets | `[GW]/api/integrations/google-sheets/auth?userId=&workspaceId=` | postMessage + polling | None |
| Google Ads | `[GW]/api/integrations/google-ads/auth?userId=&workspaceId=` | postMessage + polling | None |

OAuth detection uses dual-layer: (1) `message` event listener for instant detection via `postMessage`, (2) polling every 3s as fallback. Both cleaned up on success/timeout/unmount.

**API Key-based Integrations (4 providers):**

| Provider | Validate Endpoint | Connect Endpoint |
|----------|-------------------|------------------|
| SmartLeads | `POST [GW]/api/integrations/smartleads/validate-key` | `POST [GW]/api/integrations/smartleads/connect` |
| Apollo | `POST [GW]/api/integrations/apollo/validate-key` | `POST [GW]/api/integrations/apollo/connect` |
| Klaviyo | `POST [GW]/api/integrations/klaviyo/validate-key` | `POST [GW]/api/integrations/klaviyo/connect` |
| Mailshake | `POST [GW]/api/integrations/mailshake/validate-key` | `POST [GW]/api/integrations/mailshake/connect` |

API key flow: validate-then-connect pattern.

**Special Cases:**
- **Onesource** - UI alias for system-managed Apollo. No connect flow, no API calls. Has no backend endpoints.

### Export Flows (8 destinations wired)

All export flows use `exportService` methods via `gatewayClient`:

1. **LinkedIn Ads** - fetch accounts, ad accounts, upload audience
2. **Google Sheets** - OAuth, spreadsheet/tab selection, export
3. **Meta Ads** - accounts, ad accounts, create audience, upload data
4. **Reddit Ads** - accounts, businesses, ad accounts, create/upload audience
5. **HubSpot** - accounts, object properties, create property, export-mapped
6. **SmartLeads** - connected accounts, campaigns, sync-leads
7. **Mailshake** - connected accounts, campaigns, create-campaign, sync-recipients
8. **Google Ads** - accounts, customers, create/upload audience

Sync status write-back: all flows write to `POST /lists/{listId}/meta-columns` after successful sync.

### External AI Service

`POST https://n8n.icustomer.ai/webhook/field-mapping-agent` - external n8n webhook for AI-powered field mapping. Three-tier matching:
1. AI Agent (n8n webhook)
2. Pre-defined mappings (for known sample datasets)
3. Heuristic fallback (normalized substring matching)

### Notification Service

Novu integration via `@novu/react` v3.10.1:
- Provider: `NovuProviderWrapper` with `applicationIdentifier` (from env) and `subscriberId` (`audience-loop-{email}`)
- Custom UI (not pre-built Inbox component) because of hybrid notification system (Novu + enrichment WebSocket progress)
- Enrichment workflow IDs filtered out from Novu list (enrichment progress shown via WebSocket instead)

### Connected Account Management

- `GET [GW]/api/integrations/connected-accounts` - list all connections
- `DELETE [GW]/api/integrations/connected-accounts/:id` - disconnect
- `PUT [GW]/api/integrations/connected-accounts/:id` - edit name
- Integration connections are workspace-scoped via query key `["integrations", workspaceId]`

### Stripe (Payments)

Stripe handles checkout, portal sessions, subscription management, and metered billing for credit overage. Documented backend bugs: Stripe test/live env mismatch on portal-session, preview-upgrade, and upgrade-subscription endpoints.

### Onboarding Integration

- `GET /onboarding/progress` - checklist progress
- `POST /onboarding/complete-task` - mark task done (e.g., `connect_linkedin`, `connect_hubspot`)
- Post-connect actions are fire-and-forget (errors logged, not surfaced)
